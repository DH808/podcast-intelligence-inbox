## 00:00–04:02 开场：失败约束、内容生产、从 Anchor 到投资人的思维变化

Harry 先把 Mike Mignano 的背景放在前面：Mike 现在是 USV 的新 GP，此前是 Anchor 创始人并把公司卖给 Spotify，也在 Lightspeed 时早期投过 **Suno** 和 **Granola**。开场剪辑里 Mike 的几个核心判断先被抛出：agent 会让人类“把前所未有多的自我交给技术”；创业公司现在需要尽可能多的优势；如果他是创业公司 CEO，会在正确的地方继续最大化 token spend；USV 喜欢投“obliterate markets / existing business models”的公司；AI 产品时代很大程度上是“first + move really fast”。

Harry 问 Mike：作为前创始人，他更受“害怕失败”驱动，还是“赢的快感”驱动？Mike 的回答是两者都需要，但他非常强调**失败作为约束**：伟大的公司、产品、人往往来自约束，而失败是最极端、最澄清的约束。他回忆 Anchor 时有人问他人生中做得最好的工作在什么时候，他的答案是：**公司只剩三个月现金、即将失败的时候**。那种“如果不扭转就会死”的状态会迫使团队做最清晰、最高质量的判断。但仅有恐惧还不够，也要有一个极其雄心勃勃、看起来几乎不可能的 mission，团队需要“想赢”。他的框架是：要同时持有“怕失败”和“想赢”这两个念头。

随后两人回到 10 年前 Anchor 时代对内容形态的分歧。Mike 当年偏好更自然、对话式、让人“旁听”的录音；Harry 当年坚持更“manufactured”、更重剪辑、重包装的音频。Mike 现在承认，今天内容供给太多，工具又让普通人也能做出不错的音视频，**默认质量线被抬高了**；如果想突围，基线就必须是 excellence：强剪辑、高制作价值、Harry 当年说的 “insights per minute”。Mike 现在反而支持高制作价值的 podcast / YouTube 内容，包括多机位、多麦克风、跳出传统 studio 的拍法。Harry 补充，他很重视 time to value，不喜欢开场问“你在哪里长大”这种低信息密度问题，而会直接问“你更受失败恐惧还是胜利快感驱动”这种能立刻 hook 住人的问题。

## 04:02–08:04 为什么从 Lightspeed 到 USV：小基金、thesis-driven，与 AI 应用层机会

Harry 用玩笑方式问 Mike：从大平台 Lightspeed 到相对小而有约束的 USV，为什么做这个选择？Mike 说他来自纽约、一直围绕纽约生态，Anchor 时就认识 USV。虽然 USV 当时 pass 了 Anchor，但他和 USV 建立了很长的友谊，后来也成为 USV 基金 LP。真正吸引他的，是 USV 长期愿意**非常 opinionated / thesis-driven**，哪怕市场阶段更奖励 consensus-driven。Mike 承认，近期一些 consensus-driven 的投资确实非常赚钱，尤其 AI 模型与基础设施大公司需要巨额资本，投进去的多阶段基金赚了很多钱；但他的工作方式和下注方式更偏 thesis-driven。风险也在于：如果 thesis 错了，就会 miss 或 fail。

Harry 追问：如果未来 venture 的赢家就是 consensus-driven 呢？Mike 的回答切入 AI 周期：过去一段时间，AI 经历了一轮巨大的基础设施 buildout。OpenAI、Anthropic、xAI、SpaceX 等需要海量资本来构建新技术和基础设施，并已经创造数十亿收入、未来可能创造数万亿美元价值。但 Mike 认为基础设施已经建到一定程度，这很像互联网早期先铺光纤和宽带，再由应用层利用新技术。他的判断是：**现在进入“应用层利用 AI 基础设施”的阶段**，软件和应用会大量涌现；在这种环境里，如果不知道自己在找什么，就无法有效下注。USV 的强项正是知道自己在找什么，并在看到时敢下注。

Harry 引用 Benchmark 的 Brandon（字幕作 Mccor，疑似 Brandon 的观点）：未来 24 个月基础设施层创造/获得的价值会大于应用层。Mike 没有完全否定基础设施，他说 buildout 还没结束，基础设施层仍会创造价值；但现在已经有足够多的新技术玩具，应用层也会有巨大价值创造。

Harry 问“AI 的未来是否 always-on”，并提到 Sam Altman 的表述让他更看好 Nvidia。Mike 认为，context 对 labs 和 application layer 都越来越有价值，因此产品和公司会把用户推向 always-on 世界。他用 Alexa 的早期体验作类比：当年家里有一个设备一直听 wake word 已经令人惊讶；现在开会时有人说 “I’m granolaing this”，说明对常驻、记录、理解上下文的工具已经更习惯。Mike 的态度是：世界会朝这个方向走，for better or worse。

## 08:04–12:06 种子基金竞争：中间地带危险；早期 VC 要“发射 bat signal”

Harry 提出一个 VC 市场结构判断：5000 万到 1 亿美元规模的 seed fund 处境最差——会被大型基金用大支票和 lead round 能力压制，又不够小到能协作。Mike 没有直接同意“中间地带最差”，但给出早期基金成功条件：最好的 seed fund 非常擅长构建网络、见到优秀的人；同时要把自己的想法公开表达出来，向市场说明自己在找什么。

Mike 举例说，他刚发了一篇关于 **Rebel Alliance** 的文章，告诉市场 USV 对一个新宇宙感兴趣：open-weight models、open-source harnesses、distributed compute、agents、human-aligned agents。公开 thesis 的作用不是展示自己永远正确，而是向还没公开产品的早期创始人发出 bat signal：这就是应该找的 VC/团队。Harry 担心：如果 thesis 还没完全成型，或者自己错了，结果涌来一堆并不符合实际机会的公司怎么办？Mike 认为这没问题。投资和创业一样，都要愿意把自己放到市场里、承担风险、下注；可以用反向仓位或 counter-position 来对冲，但不能因为怕错就不表达。

## 12:06–18:12 模型层未来两条路径：递归自我改进 vs S-curve 平台化；open / routing / harness 的角色

Harry 进入预测环节，问五年后的模型市场结构。Mike 给出两条可能路径。

第一条路径是某些接近 superintelligence 的实验室一旦达到递归自我改进（recursive self-improvement），就会进入指数式智能增长，先达到者可能不可追赶。Mike 解释 recursive self-improvement 是指超级智能 AI 能做自己的 AI research，部署后持续改进自己，直到某种未知约束让它 plateau。约束可能来自 transformer 架构无法继续扩展、可用数据不够、或硬件基础设施跟不上。Mike 不希望这是最终路径，但承认这是一个可能性。

第二条路径是 AI 像历史上多数技术一样遵循 S-curve：慢启动、指数增长、然后因某些限制 plateau。如果智能 plateau，模型技术会更像 commodity，其他 labs 有机会追上来，竞争焦点会转向价格、产品体验和 intelligence stack 的各组成部分。

Harry 追问：如果真有递归自我改进，是今天的 OpenAI / Anthropic 等领先模型公司完成，还是新公司完成？Mike 认为大概率是已经拥有 compute、frontier models、基础设施、芯片优势的 labs。但他也保留新架构 leapfrog 的可能：有很多优秀研究者在研究可能优于 transformer 的新架构，例如 Thinking Machines、Safe Superintelligence（Ilya / Daniel）这类团队。

在第二条 S-curve 路径下，Mike 认为企业和个人会优化两件事：第一是成本，即 intelligence 和 spend 的 trade-off。企业会使用 open-weight / open-source models、routing layer，把任务路由到最有性价比的模型，而不是永远 token-maxing 选择最强模型。第二是选择更 human-aligned 的工具/产品/“harness”。

Harry 问什么是 harness。Mike 的定义是：**和模型紧密耦合的应用**，形成 model-product loop。例子包括 Claude desktop / Claude Code / Claude Co-work，也包括 Hermes（字幕提到可接管 Mac Mini 的 harness）和 Arendelle 的 Pi。Mike 认为，未来用户会更在意 agent / harness 的 incentives 是否与自己一致。大模型公司的模型天然有激励让 lab 的模型更聪明、更快、更好；但当用户把 agency、个人信息、目标、信用卡都交给 agent 时，会问“这个 agent 到底为谁工作？”这也是他之前在 X 上写 “who is your agent working for?” 的核心。

## 18:12–24:16 Agent 托付、隐私与 token spend：大公司会控费，创业公司应在 coding 上 token-max

Harry 认为人们会像接受线上信用卡、Apple Pay、线上婚恋一样，很快接受把钥匙交给 agent；他直言除了 hardcore technologists，没人真正在乎 privacy。Mike 承认历史上互联网和个人计算的发展说明 Harry 可能是对的，但他强调这次不同：以前 Chrome 或网页没有替用户买东西、给亲友发极私人的消息，也不是“第二个自己”；agent 将更像 second self，因此至少一部分用户会更在意模型/agent 的 incentives。Mike 不要求所有用户都关心 human alignment，也不要求每个 agent 都 superhuman aligned；只要有足够多用户关心、足够几个 good actors 存在，就能形成市场力量，约束其他玩家。他承认历史上这未必会发生，但认为可能。

Harry 问 Anthropic 的 Dario 是否在 marketing / messaging 上 own goal，因为他长期讲 labor displacement、客户会失业。Mike 的回答相对防守：他看到的是一家极度 mission-driven 的公司，而 mission-driven companies 可以非常成功；不管外界喜欢与否，Anthropic 已经非常成功，录制时可能是全球最有价值的私营公司。

Harry 接着把问题落到 Anthropic 收入增长的关键变量：开发者工资中用于 tokens 的比例。他引用 Marc Benioff：Salesforce 在 Anthropic 上花了 3 亿美元，针对 dev team，相当于 dev salaries 的 3.8%。如果这个比例升到 20%，Anthropic 被严重低估；如果到 100%，市场比想象大得多；但如果比例维持低位，或迁移到 open models，就完全是另一种游戏。

Mike 认为 token spend 是真实风险，但要区分两类客户：

1. **大公司/ incumbents**：Salesforce、Microsoft、Meta、Uber 这类组织员工数量巨大，如果每个员工 unlimited token spend，业务 fundamentals 会受压。因此大公司会开始约束 spend。
2. **创业公司**：组织更小，更容易控制总额；同时创业公司需要一切优势。因此如果 Mike 是创业公司 CEO，他会在“正确的地方”继续最大化 token spend，尤其 coding。他会把 summarization、ops 等简单任务交给更便宜的模型（如 Sonnet 级别），但 coding 时希望用 frontier model，争取对 Salesforce 这样的 incumbents 的优势。

Harry 进一步问，最好开发者会去一个 token budget 被限制的大公司，还是去一家说“free-for-all, be your best self”的创业公司？Mike 认为会去创业公司，尤其 mission-driven 的创业公司。他补充，过去几年资本太充裕，很多公司不需要 mission-driven；现在 mission 会重新变重要。

## 24:16–28:20 100x engineer、open models、routing layer：机会存在，但商业模式未定

Harry 提到工程组织可能演变为少数 100x engineer，用高 token spend 替代 10 个普通工程师。Mike 同意工程组织会变化，但不确定最终形态；他猜测会出现更小、更高质量、更高 caliber 的工程团队，因为低阶任务会被 agents 代理。

在 open ecosystem 上，Harry 问企业工作流中有多少可以用 open models 完成。Mike 的估计是：**企业里 80% 非 coding 任务可以用非 frontier 模型完成**。例如总结、文档、brief 生成等不需要 frontier；coding 则更可能需要 frontier models。Mike 观察到 open-source / open-weight 模型追赶速度比以前快，因此非 coding 场景未必需要站在 frontier。

Harry 指出中国开源模型发展速度惊人，问是否令人担忧。Mike 的回答从激励出发：startups 和 teams 会去激励所在的地方；如果 Rebel Alliance 真有机会，会有越来越多聪明团队转向 open。

随后两人讨论 **routing layer**。Harry 问会不会出现 100–500 亿美元的 routing 公司，还是 inference providers（如 Fireworks、Base10）或上层公司（如 Nebius）吃掉这一层。Mike 认为 routing 现在重要，因为企业要优化 token spend，既要任务能力匹配，也要成本最优。会有 OpenRouter 这样的公司专注 routing，也会有 stack 其他层的公司把 routing 作为 monetization 机会。许多公司在讲商业模式时都会提到自己要建 routing layer。

商业模式问题上，Harry 质疑 routing 很容易变成 commodity pipe。Mike 说很多公司目前想在 token / inference 价格上加 small margin，但这可能不是好商业模式。他听到一个有趣想法：routing layer 可以采用 bounty model，只有当它为某任务选择了最高效或最合适模型时才收费。Mike 认为这个 idea 有趣但还没看到成熟实现。Harry 仍认为单独 routing 很难成为 500 亿美元公司。Mike 的反驳是，企业基础设施公司一旦嵌入 developer workflow / ecosystem、成为 gold standard，就很难被替换；routing 也可能出现这种“每个开发者都采用且不想移除”的层。

## 28:20–32:24 USV 没投模型层？转向能源、算力位置与“资本主义解决约束”

Harry 问：说 USV 错过 model game 是否不公平？Mike 说 USV 玩的是不同游戏，这也是 USV 的历史。USV 也许没投大模型层公司，但过去几年一直在投 **energy**。他的逻辑是：不管哪个模型赢，只要相信 AI 和 intelligence，就需要底层能源层支撑。随着时间推移，市场发现需要的能源比想象更多，且需要更强的能源可移动性/可接近性。Mike 先口误说 USV 从 2001 年开始投能源，随后修正为 2021。

Harry 问能源投资资本开支很重，venture 怎么投。Mike 的框架是：能源当然 capex-intensive，但创新最早期常常不是特别资本密集；venture 应该在早期团队还在做 science experiments、别人还没想到时下注。例子包括 USV 投资的 **Radiant**，做可从工厂线下来的小型核反应堆，并将成为美国最早进行相关测试的公司之一。

Harry 又提到 Boom Supersonic、AI turbines 的机会，并说自己投了 **Rune**。Mike 说 Rune 做的是把微型数据中心放在发电机、风场等能源旁边，解决“如何让能源尽可能快、尽可能近地靠近 compute / data centers”的 portability 问题。Harry 还提到 **Panthalassa** 做海上数据中心，以及 Elon 可能把数据中心建到太空。他把这看作 Adam Smith invisible hand / 资本主义的乐趣：市场看到能源危机，就出现 Rune、Panthalassa、space data centers 等方案。Mike 认同这是非常有趣的投资时点。

## 32:24–36:25 “Don’t automate, obliterate”：AI 应用如何抵抗模型层侵入

Harry 对欧洲许多创业者仍在做 SMB accounting 表示不满。Mike 的回应区分“automation”与“obliteration”。他承认 enterprise automation 也有机会，但 USV 更喜欢投能“obliterate”的公司——不是简单让现有工作流更快，而是重构市场和商业模式。USV 甚至不做太多传统 enterprise investing，因为它常常是卖给中间人、让现有企业更有效率；USV 更想投“彻底重新发明做事方式”的公司。

例子是 **Doctronic**：USV 几年前领投 seed，当时想法看起来疯狂——用 AI 把医生放进每个人口袋。Mike 说，他们本可以投让医疗实践更高效、帮处理保险 claims 的工具；但 Doctronic 的想法是把 doctor in everyone’s pocket，彻底重构模型。

Harry 提出反例：在教育和医疗里，横向平台往往是最大赢家，比如 YouTube；ChatGPT 的第二大用途可能也是 health。问题是模型提供商是否会吃掉应用层。Mike 用 **Abridge** 回答：Abridge 解决类似问题已经接近 10 年，在高度监管的医疗行业，不能说“我们现在做 healthcare”就进来；要建立关系、伙伴关系、通过监管障碍，这些会成为 moat。USV 2018 年投 Abridge，经过五六七年才到如今在大量医生和医疗系统使用的拐点。

Mike 的更广义判断是：每代技术都会有人担心 labs / hyperscalers / incumbents 吃掉所有应用，但总会有 startups / founders / entrepreneurs 专注并提前做艰难事情而胜出。Spotify 对抗巨头就是 David vs Goliath，沿途有 50 种方式可能被杀死，但通过 specialization 和 early/often 做 hard things 最后成为赢家。Mike 也坦言，如果 VC 不相信 startups 能打败 Goliath，这个行业就没意思。

Harry 补充，竞争也未必二元：即使 Anthropic 派整个团队做 design 对抗 Figma，Figma 仍是几十亿美元收入的公司，市场还有多个玩家。在 dev landscape 上，Harry 过去以为会出现 runaway winner，但实际上 Claude 很强，Lovable、Cognition、Replit（字幕作 Rapid）、Cursor 等也都很强。Mike 同意：多数市场不是赢家拿 80% 或 100%，常见的是第一名拿 30%，剩下 70% 仍可产生很多大赢家。

## 36:25–42:31 Granola、企业进入路径、Microsoft bundling 与 AI 产品的上下文 moat

Harry 问 Mike 在 USV 看到的 deal flow 是否和 Lightspeed 不同。Mike 说，如果你持续向市场输出 thesis，就会看到匹配这些 theme 的交易。USV 小基金必须更聚焦、更有约束，例如现在聚焦 energy 和 Rebel Alliance，因此会看到相关公司主动来找他们。

Harry 问 Mike 当前不确定和担心什么。他提到很多人担心 model providers 侵入 application layer，并举自己投资的法律 AI 公司 Lagora，Anthropic 公开说想进 legal。Mike 承认，即便他说 startup 总有机会，竞争威胁仍真实存在。他在 Lightspeed 时与 Granola、Suno 合作密切，这两家公司都面对巨大 incumbents；Granola 尤其面对 OpenAI 直接推出竞争产品、Notion 这样资金充足的 startup 也推出竞争产品。

Harry 追问 Granola 的真正难点是否是 enterprise penetration：很多 VC-backed 产品可以做到 5 亿收入，但要到 50 亿收入就难。Mike 认为 Granola 的优势之一是**极度聚焦**：它只做 notes。Harry 认为它最终可能要做更多；Mike 的路径是先用一个明确、最好用的产品进入企业，“我们只做你的 second brain / meeting notes，而且是最好的”。如果一上来就横向很宽，就要让企业放弃 Gmail、Google Docs、Sheets、Calendar 等一堆东西，阻力太大。先 foot in the door，再横向扩展。

Harry 问 Microsoft bundling 的威胁：Microsoft 历史上常用“65–70% meh product + bundle”打败竞品，比如 Teams / Copilot。Mike 承认担心，Slack 就经历过这种冲击，但 Slack 仍是 270 亿美元级别的 valuable product。对 Granola / AI 产品，Mike 学到的是：**context 极其有价值**。如果 Granola 进入组织、所有人开始用它，组织会积累大量富含上下文的信息历史。这不是企业愿意轻易放弃的东西。AI 产品时代的关键是 first 和 move really fast，尽快积累不可迁移的 context。

Harry 担心一个相对“中等”结果的问题：当 SpaceX、Anthropic、OpenAI 级别的 trillion-dollar outcomes 出现时，一家做到 5 亿收入的公司会显得 meh。Mike 回到 fund strategy：这取决于你优化什么、你的 mission 和 constraints。USV 一直是小基金，这是他们喜欢玩的游戏；小基金可以在自己的目标下非常成功，未必适合 100 亿美元基金。Harry 也说自己如果没有 media 就不会做 venture，并告诉 LP：今天没有 4 亿美元基金就做不了 Series A，因为 Series A 估值可能 8000 万、1 亿、1.5 亿 post。

## 42:31–48:35 所有权、价格、基金规模：早期拿 ownership，后期看 cash-on-cash

Harry 问 Mike 是否对 ownership 更有弹性。Mike 分阶段回答：seed 和 Series A，ownership 仍重要，因为今天赢家会很快融很多钱、估值迅速升高；对小基金来说，后面 rounds 很难再买到 ownership，所以必须在 seed / A 阶段拿到。但一旦公司跨过某个 chasm 到 Series B/C，被识别为市场领导者、潜在 generational company，ownership 就不再是核心，而是 cash-on-cash underwriting：投 2500 万美元进去，可能是多少倍？10x、100x、1000x？如果足够大，不必执着百分比。

Harry 提到双方刚共同参与 **FOMO**（Paul 创办，下一代 trading app）的 later round。Mike 说这是后期不看 ownership、看市场空间和 multiple expansion 的例子：他们认为 FOMO 还能大很多，因此愿意投一张支票；USV 受基金规模约束，未必每轮都能跟，但从当前 upside 看值得。

Harry 问：如果不认为是 mega outcome，还该投吗？Mike 说对 USV 后期投资来说，如果不是巨大 outcome，可能不值得；早期也希望市场大、serviceable area 大。小基金相对大基金有更多 downside protection，因为一个 2.75 亿美元核心基金比数十亿美元基金更容易倍增。

Harry 认为现在可能是大型平台基金最好的时代，因为 outcome size 扩张让它们能在 growth vehicle 上做出 venture-like returns，Thrive、OpenAI 投资就是例子。Mike 承认，大型基金在资本密集 AI 基础设施阶段表现出色，因为它们能投进需要海量资本的公司。但他也认为接下来应用层会出现大量更不资本密集的 startup；这些机会需要知道自己在找什么，可以用更 modest check size 进入。Mike 的市场结构观点是：中间基金最难；要么很大、玩 consensus game，要么很小、很 opinionated，用小基金倍增。

价格方面，Harry 说自己过去一年有两笔 deal 被 2–3 倍价格 outbid：自己出 1.5 亿 post，对方出 3 亿；另一次对方 4 亿。问是否该直接付。Mike 引用 Fred Wilson 的最大教训之一：**never pass on price**。但他马上加 nuance：对 USV 来说，后期如果坚信是 market winner 且 multiple 很高，可以对价格更有弹性；最早期用小基金投时，价格仍有上限，否则 fund math 不成立。

## 48:35–54:40 投资人 lesson：别把自己的想法投射到 founder；founder > market > product

Harry 引用 Peter Fenton 的一句话：用价格作为 conviction 的 litmus test。如果某些 founder 价格翻倍你仍愿意投，说明 conviction 真强。Mike 认同，回看自己最好的 deal，有些即使价格翻倍也愿意投。

Harry 问 Mike 做 VC 最大教训。Mike 的第一条是：**不要把自己的想法投射到 founder 身上**。作为前 operator，很容易觉得“我知道这个业务该怎么建 / 产品该怎么做”，但这非常危险。他提到 Lightspeed 前同事 Jeremy Liew 曾提醒他：你在作为前创始人投射自己的判断。即使投资人想法是对的，公司终归是 founder 的，他们可能想用完全不同方式构建。如果你因为自己假设的 A/B/C/D 路径而下注，但团队根本不这么做，那你就是下了错误的赌注。投资人必须相信团队自己的判断和执行。

Harry 问 founder / market / product 的排序。Mike 说刚做 VC 时会排 product > market > founder，现在完全反过来：**founder > market > product**。早期公司大多会 pivot，最重要的是 founder 是否 resilient、能否 execute、能否 adapt to change。

Harry 问做错 founder read 时通常漏看了什么。Mike 说常常是 **communication**。沟通触及公司建设每个部分：招聘时要表达 mission / values / why exist / what to accomplish；融资时要向投资人讲清楚；产品愿景要传达给团队；市场故事也要讲好。回看 evaluation mistakes，communication 是最大因素之一。

Harry 自曝 miss：Suno 和 Granola。Suno 当时 David Frankel / Founder Collective 在 seed 介绍给他，但因为只能投 20–25 万美元、ownership 只有 1%，Harry 被“LP 要高 ownership”的框架束缚而放弃；Granola 则是缺乏想象力，觉得 Chris 很好但想法太未成型。Mike 对两者的投资逻辑不同：Granola 是纯 founder bet，因为他认识 Chris 15 年以上，在 Anchor 期间 Socratic（Chris 前公司）的办公室就在后面，他们一起办 meetup、创始人之间交流挣扎和建议，Mike 亲眼见过 Chris 能做事；Suno 则是 thesis-driven bet，源于 Anchor 时对“产品如何 democratize a creative medium”的理解。音乐在 AI 前从未被真正 democratized，因此 Mike 去见了所有做 AI music 的团队，最后遇到 Mikey / Suno 团队，先由 thesis 导向，再叠加 founder lens。

## 54:40–58:43 Suno：从工具/平台到“creative entertainment”新行为

Harry 问，见到一个 founder 想投是否通常很明显。Mike 说不总是，但 Suno 的 Mikey 非常明显。他们在 Hoboken 一家餐厅见面，Mikey 家在 New Jersey，双方立刻建立连接。Mike 同时感受到团队对模型和世界愿景的理解，以及 Mikey 作为前音乐人对“让音乐创作/表达/创造力更容易”的深度连接。

Harry 问在 50 亿美元估值投 Suno 时如何 underwriting。Mike 说他们认为 upside 可以“loosely speaking, unlimited”。历史上真正 democratize 一个 medium 的互联网平台，如 YouTube、TikTok、Twitter、Facebook（写作/出版某种程度上），都是极其有价值的。Harry 问 Suno 是否必须从 tool 变成 platform。Mike 说投资时大概会回答“是”，当时可能认为它需要 creator side 和 consumer side，类似 Spotify 的双边结构。但现在他不确定当初是否理解了 Suno 的真正形态。

现在 Mike 更相信 Suno 团队所说的：它是一种新行为，团队称之为 **creative entertainment**。用户进 Suno 做音乐，不一定为了发行、分发、变现，而是为了创作本身的乐趣。Anchor 时代用户做 podcast 目的通常是 distribution 和 eventually monetization；现在 AI 让音乐创作本身足够好玩，人们是“making music to make music”。Mike 把它类比 Claude Code、Midjourney，或用 AI 做游戏的平台：用户未必想成为职业游戏开发者，只是在电脑上做游戏很快乐。

## 58:43–01:02:46 Substack、独立媒体和 quick fire：传统媒体仍在被拆包

Harry 抓住 Mike 没回答 miss，Mike 说自己遗憾没投 **Substack**。虽然不是传统 AI 公司，但过去 8 年，趋势越来越明显：人们走向 self-publishing，控制自己的 media destiny、创作、发布、变现。除 X 之外，Substack 可能是做得最好的公司。Mike 喜欢 founders、purpose、product、platform，但没有机会投。

Harry 问传统媒体是否死了。Mike 说 traditional media 在很多方面已经死了。他承认自己低估了 independent media 能有多大。Anchor 当年愿景是 democratize audio / give everyone a voice；到 2022 年离开 Spotify 时，他一度以为机会已经 baked：YouTube、Spotify、X 都有了。但过去四年机会又变大很多。每个传统电视节目、媒体人都在迁移到 self-publishing / independent media；很多大媒体组织的知名人物离开，自己做 YouTube show 或 Spotify show。Mike 认为 television 仍在被 massively unbundled，还没有到顶。

Harry 讲自己曾在 TechCrunch Europe、Wired Europe 关闭后，想拿 1000 万美元聚合欧洲最好的 writers 做下一代媒体，但投资团队问他：既然没有 editorial control，就像 OpenAI 不能控制 TBPN，一起做的意义是什么？Mike 回应 editorial freedom 很重要，这也是他喜欢 Substack、X、YouTube、Spotify 的原因。

Quick fire 开始后，Harry 问最好的第一次 founder meeting。Mike 说是 **Bren Putnam / Board**（字幕作 Bor/Board），桌面游戏主机公司，USV 刚宣布 Series A。Bren 之前创办 Mirror 并卖给 Lululemon，拥有硬件、供应链、产品的深 domain expertise。Mike 见完后觉得必须投。

## 01:02:46–01:08:49 快问快答：Factorial、Haystack、Nat Friedman、Lightspeed Growth、Fred Wilson 与 parenting

Harry 问如果只能投一个 seed fund，Mike 选谁。Mike 首先提 **Matt Hartman 的 Factorial**：它不是传统 scout 模式，而是给真正的 angel investors 额外资本，让他们在自己的钱之外再投资，Factorial 通过 carry 等方式变现。这个网络里有 Hugging Face 的 Clem 等真实 angel。Mike 认为这个模型很聪明。第二个并列答案是 **Haystack**，Mike 非常欣赏 Semil、Divya 的工作和他们已产生的 generational outcomes。

谁发来的 deal signal 最高？Mike 提到在 Lightspeed 时与 **Nat Friedman 和 Daniel Gross** 密切合作；Nat 发来的 deal 他一定重视。Suno 的 Mikey 也是通过 Nat 认识的。Harry 感叹 Nat/Daniel 不再投资是 venture 行业巨大损失。

喜欢哪个 growth fund？Mike 说 **Lightspeed Growth Fund** 很强，过去几年做了 Anthropic、xAI、SpaceX 等 AI 相关投资，表现 phenomenal。

与 Fred Wilson 工作最大 lesson？Mike 说最重要的是与 founders 的关系。一方面，投资人最能帮助 founder 的方式就是在他们身边，因为创始人是孤独、惩罚性的工作；有一个能信任、利益一致的人很重要。另一方面，自私地说，reputation 在 venture 里就是一切，而好 reputation 来自与 founders 的良好关系。

Harry 问 parenting advice。Mike 说自己成长时父母非常支持他的兴趣，但不把他压向某个方向。家里很重视 baseball，父亲也是 former coach / player；Mike 没继续打棒球时，父亲只是问“那你下一步想做什么？”而不是施压。他希望对孩子也这样：支持、开车送去训练、赋能，但不施压。

## 01:06:49–01:10:50 过去 12 个月改变最大：大公司不能做一切；未来 5–10 年的乐观目标

Harry 问 Mike 过去 12 个月改变最大的一点。Mike 说，虽然前面讲过模型公司不会吃掉一切，但不久前他也曾认为 model providers 可能做所有事情，世界会由一个或五个公司统治。过去一两年他被提醒：**公司不能做所有事，即使最大公司也不能**。十到十五年前大家也这样看 Google、Apple，以为它们会拿走一切，事实证明不能、也不会。Harry 补充，Rory O’Driscoll 也提醒他，Microsoft 曾经一度考虑成为银行，说明 incumbents 总会看很多方向。Mike 承认有时这会成功（AWS 是例子），但总体上不能做所有事。

最后 Harry 问未来 5–10 年最兴奋什么。Mike 说他和 USV 伙伴 Rebecca、Nick 讨论过自己想在 USV 做什么，排序是：

1. 每天和自己喜欢、享受合作、能一起开心的人工作；
2. 与 incredible founders 合作，建立关系和动态，帮助他们改变世界、建立 generational companies；
3. 为基金创造 generational returns。

这个顺序很重要。Mike 认为如果优先优化 fun 和 day-to-day 的享受，好事会发生。

## 投资研究摘记：可验证的主要 claims / 线索

- **AI 周期判断**：基础设施 buildout 仍未结束，但应用层类似互联网宽带/光纤之后的应用爆发阶段，机会会大量出现。
- **Rebel Alliance thesis**：open-weight models、open-source harnesses、distributed compute、agents、human-aligned agents 是 USV 正在公开吸引的方向。
- **模型市场两路径**：递归自我改进导致 winner runaway；或 S-curve plateau 导致模型商品化、价格/体验/stack 层竞争。
- **token spend 分化**：大公司会控费；创业公司尤其 coding 应继续最大化 frontier token spend，因为这是对 incumbent 的优势。
- **open model 使用边界**：Mike 估计企业 80% 非 coding 任务可由非 frontier/open 模型完成；coding 仍偏 frontier。
- **routing layer 商业模式未定**：简单加价可能 commodity；bounty model 或嵌入 developer workflow 成为 gold standard 才可能更大。
- **energy / compute locality**：AI 底层约束从模型扩展到能源和能源可移动性；Radiant、Rune、Panthalassa 等是相关样本。
- **应用层 moat**：监管关系（Abridge）、上下文沉淀（Granola）、聚焦切入、速度、品牌/信任、专门化做 hard things，是抵抗 labs / hyperscalers 的路径。
- **VC 策略**：基金结构要么大而 consensus，要么小而 thesis-driven；中间地带危险。早期重 ownership，后期 market winner 可转向 cash-on-cash。
- **founder assessment**：Mike 的排序为 founder > market > product；最大风险是把投资人自己的产品/业务设想投射到 founder 身上；communication 是 founder read 中容易低估的变量。
