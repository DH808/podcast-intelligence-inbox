## 核心定位

这是一场 28 分钟的现场对谈，信息密度集中在四个问题：AI Engineer 为什么成为独立职业；frontier lab 与政府的关系是否会改变发布节奏；当前 LLM 能否递归自我改进到 AGI；模型无关路由型 agent 公司是否比垂直一体化 lab 更有优势。

swyx 的立场并不迎合主流叙事：他认为 **AI Engineer 的长期机会来自模型峰值能力与真实产品部署之间的“白色空间”**；但他不认为当前 LLM 靠递归就能发现真正的 unknown unknowns，也对“多模型路由天然优于单模型深度集成”保持怀疑。

## 00:00–04:30｜AI Engineer 为什么会成为一个独立职业与会议品类

Matthew 先问 AI Engineer Conference 的起源。swyx 说，他长期做技术会议演讲，见过前端工程、云工程、数据工程分别从一组实践演化成有专门技术栈、会议、意见领袖和职业身份的领域。AI 明显会重复同一路径，因此他买下 `AI.engineer` 域名、写出定义性文章并规划会议；Andrej Karpathy 的公开认可帮助概念起飞。

他没有独自完成会务。合作伙伴 Ben 运营 Reactathon，负责物流，swyx 负责内容；基本做法是把自己多年参加会议时见到的最佳实践拼起来。第一届最难的是让人相信一个没有历史记录的新会议值得参加；如今已有口碑和 YouTube 内容，证明成本下降。Gartner 两年前已把 AI Engineering 放到 hype cycle 顶部，但热度没有像典型短周期概念那样迅速消退。

**机制**：每次基础平台跃迁都会创造一层“把新能力工程化”的专业岗位。模型研究解决能力上限，AI Engineer 解决可靠性、评测、产品集成、成本、延迟、数据和用户体验。

## 04:30–10:00｜新一代模型：极聪明但慢，适合“只给最难问题”

对谈转向 Anthropic 新模型是否被“削弱”、是否发生误拒答和降级路由。swyx 明确说没有硬数据证明发布后被 nerf；false refusal 确实烦人，但他个人尚未遭遇被路由到低档模型的情况。

他的使用感受是：模型正常工作时“极其聪明”，但也非常慢，所以不会用于所有任务。合理产品策略反而是只把最需要智力的任务交给它。这与 frontier lab 自己的资源配置目标一致：最昂贵模型服务高价值问题，普通请求走更便宜路径。

两人也质疑此前“Anthropic 因芯片/算力短缺而无法发布新模型”的流行叙事。swyx 认为，模型公司知道自己的路线图，外部观察者往往把短期供给和发布选择过度简化为单一算力约束。

## 10:00–14:30｜政府介入模型发布：统一联邦框架可能优于州级碎片化，但创新远未结束

Matthew 问 frontier lab 与政府互动是否变成“pay to play”或“bend the knee”：企业给出经济利益，换取更快发布或放松监管。swyx 避免对具体政治交易作无证据判断，并提醒这不是一次性博弈；美国每四年选举，企业若只优化当届政府，可能忽略长期多轮关系。

对于政府要求模型延迟几周发布、先做安全审查，他认为白宫大概相信自己在执行一致框架。即便不完美，联邦层面的统一规则也可能优于各州分别推出不同法案。

他反对在技术周期早期把 frontier AI 当成熟公用事业实行成本加成式监管。历史上的电力、照明等行业也经历长时间创新后才进入更稳定监管阶段；ChatGPT 出现仅三四年，就断言行业创新耗尽、只能交给自然垄断并不合理。AI 仍可能需要数十年演化。

## 14:30–18:30｜p(doom)、AGI 时间线与“我们未必活在历史终点”

被要求给出 10 年和 50 年 p(doom) 时，swyx 对 10 年接近零；50 年先随口考虑 10%，随后下调到约 **5%**。这不是模型化概率，而是现场表达。

他的核心理由是反对“主角谬误”：人类容易假设自己恰好活在一切结束的时点。也可能当前 LLM 走到瓶颈，随后出现 30 年 AI winter；也可能真正关键的新范式在几千代以后。未来人回看今天，或许仍把它视为史前阶段。

他借《Foundation》和《Dune》说明，讨论 AI 风险应拉长文明尺度，而不是只在当前模型代际外推指数曲线。

## 18:30–21:30｜当前 LLM 能递归自改进，但不等于能走到 AGI

Matthew 提出一个更精确的问题：LLM 是否足以触发 recursive self-improvement（RSI），再由 RSI 找到下一代架构并最终实现 AGI？

swyx 的答案分两层：

1. **LLM 已能做有限 RSI**。会议甚至有 auto-research 专题，模型可以生成实验、改代码、评估结果再循环。
2. **但递归范围受已知分布限制**。它多半探索人类已探索过、训练数据中已有的区域，离真正发现 unknown unknowns、产生原创科学范式或完整 world model 仍有距离。

因此，他对“现有 LLM 通过更多循环自然跨越到 AGI”的回答偏否定。即使无限 token / 无限预算可以把当前模型的可用性推到极致，最后仍可能需要不同架构。对谈中举到 Thinking Machines、Together AI、SSM 等方向作为可能探索，但没有断言任何一家已解决问题。

**投资含义**：agentic loop 会继续创造近中期价值，但不能把“更多 test-time compute”直接等同于“自动发明下一架构”。研究平台、实验自动化、world model、非 Transformer 或混合架构仍有独立期权价值。

## 21:30–24:30｜Capability overhang：AI Engineer 的工作为何长期存在

Matthew 认为，即使上一代模型（节目中提到 Opus、GPT 5.5）仍有巨大未兑现价值，新一代又带来更大 capability overhang。swyx 用一句话定义 AI Engineer：其工作存在于“模型峰值能力”与“把能力部署到所有真实产品”之间的白色空间。

模型研究往往只在某个 benchmark 或任务上形成尖峰，能力不会自动均匀进入每个产品。工程师要补齐评测、工作流、UI、数据、权限、可靠性和成本，所以只要研究继续制造新的能力峰值，工程层就持续有工作。

但这个市场会周期性“春季大扫除”：围绕旧模型限制搭出的复杂脚手架，下一代模型可能用一个 prompt 就完成。工程团队不应依恋代码；目标是更便宜、更快、更容易地服务客户，旧抽象失效就应删除。

**对软件公司的含义**：护城河不能只是临时 prompt chain 或补偿模型缺陷的胶水。更稳固资产是客户分发、专有工作流、反馈数据、权限/合规集成和对结果负责的系统。

## 24:30–28:04｜多模型路由公司会赢吗？swyx 对“模型无关”营销话术的反驳

Matthew 提出当前最热门的 bull case：token maxing 对普通用户不可负担，因此 Cursor、Cognition、Factory 等非 frontier agent 公司可保持模型中立，按任务路由，且比只推自家模型的 frontier lab 更有动力优化成本。

swyx 承认这是合理结论，也是市场当前常见说法，但不表示他认同。技术史上最大的成功者往往“all in on one thing”：云时代有公司深度绑定 AWS 或 GCP，也有 Terraform 式多云路线。路由型产品的问题是：为了兼容每个模型，容易退化为所有模型的最低共同能力，无法利用某个模型最独特、最新的功能。

单一平台深度集成还有规模经济：AWS 控制整套栈，可以做端到端优化；agent lab 虽能从多个模型供应商取得折扣，但比较成本时不能拿公开 API 标价直接对比 Anthropic 给大客户或 agent lab 的合同价。

因此，“model routing”有时更像营销口号，而不是自动成立的结构性护城河。真正的问题是：
- 路由节省的成本是否大于放弃最新原生能力的损失；
- 产品是否能持续跟上各模型不同的工具调用、上下文、缓存和 agent 特性；
- 深度绑定单一模型带来的能力领先，能否抵消供应商依赖；
- 客户更重视最优质量、最低成本，还是可迁移性。

# 研究结论

1. **AI Engineer 是能力扩散层，不是模型研究的替代品**：capability overhang 越大，工程机会越多；但纯胶水会被新模型周期性清除。
2. **RSI 与 AGI 必须拆开**：当前模型已经能递归做实验和代码改进，但在已知分布内循环，不等于发现未知科学或自动产生下一架构。
3. **多模型路由没有免费午餐**：模型中立降低锁定和成本，但可能牺牲单模型深度、端到端优化与首发能力。投资时需看实际任务级质量/成本数据，不能只听“model agnostic”叙事。
4. **监管仍在技术形成期**：统一联邦框架可能减少碎片化，但把行业按成熟公用事业监管为时过早。
5. **swyx 的风险判断偏低且时间观很长**：10 年 p(doom) 接近零、50 年约 5%只是现场主观口径，重点是反对把当代人视为历史终点。
