# 第一部分：Andrew Feldman / Cerebras

## 00:00–03:00｜AI 基建建设规模：不是普通云扩容，而是全球资源动员

**主持人问：2026 年 Cerebras 在做什么？德州等地的数据中心建设到底有多大？**

Feldman 的回答从物理尺度开始：这轮 AI 建设与过去的软件或单机硬件周期不同，讨论的是“建筑物、城市级电力和跨国部署”。他称，未来若干年的数据中心用电会达到极大规模；单体建筑可能有数个橄榄球场大小，输入电力高于一座中型城市。

建设并非只集中在美国。美国、加拿大、北欧、法国及欧洲其他地区、中东都在建；连过去并不处在全球计算版图中心的哈萨克斯坦、塔吉克斯坦、格鲁吉亚、亚美尼亚也在推进。Feldman 的判断是，几乎每个国家、美国每个州都认为自己必须参与。

需求方则是 OpenAI、Anthropic、Google、Microsoft、AWS，以及主持人口中的 SpaceX/xAI 等。其共同特点不是试探性采购，而是“insatiable”：芯片和容量在完工前就预订，数据中心建设追赶的是昨天已经出现的需求。

**投资含义**：AI 基建需求呈现全球化和主权化，而不只是美国 hyperscaler 的集中采购。瓶颈会沿着电力接入、土地、冷却、并网、施工、先进封装和推理硅扩散；但 Feldman 是供给方 CEO，其需求口径应与订单可取消性、交付周期和客户集中度交叉验证。

## 03:00–05:30｜250 亿美元 backlog：行业在追赶“已预订需求”

Feldman 称 Cerebras 有约 **250 亿美元 backlog**，并强调公司并非孤例。OpenAI、Anthropic、Google、Microsoft、AWS 都想要更多数据中心。现在不是传统的 “if you build it, they will come”，而是需求已经订出，供给方要防止客户因为拿不到容量而离开。

主持人追问：如此大的 token 消费是否真的创造价值？Feldman 区分“净价值巨大”和“存在大量实验”两件事。早期 AWS 也让工程师绕过内部 IT，用信用卡自行开资源；其中很多非常有用，也有部分纯属浪费。AI token 亦然：存在试错不等于整体价值不存在。

他用 Costco 类比：刚开业时，消费者像逛 Safeway 一样逐条过道购买，最后带回许多不需要的整包装商品；成熟后只去买烤鸡和生日纸杯蛋糕。企业最初也可能给所有人不受约束的 token，随后会分层：对高生产力人员放开预算，对普通任务改用开源或更便宜模型，并形成真正会部署 AI 的 systems thinker。

## 05:30–10:00｜从 prompt 工程转向“理解意图”；企业将按任务分配模型与 token

主持人观察，创业者、VC 和研究员在使用 agent 后，工具反过来要求使用者说明目标、系统边界和需求文档。Feldman 回忆早期计算机“严格照你说的做”，最初 prompting 也一样，微小措辞会导致巨大输出差异；新模型则开始补足未表达的意图，例如主动建议图表同时使用折线和柱形，而用户事前并未精确要求。

主持人以 Hermes Agent 为例：他把“每小时找出尚未被别人识别的全球趋势”设为目标，观察模型自行讨论应查 Hacker News、Reddit、社交平台和 Instagram。Feldman 将这定义为 reasoning model 在展开工作，而不是传统的搜索摘要。

这里的核心不是“无限 token 自动等于价值”，而是企业会逐步建立路由和预算纪律：
- 高难度、差异化问题给最强 frontier model；
- 常规数据搬运和企业流程交给便宜、可靠、可控的模型；
- 对能把 AI 转化为生产率的人增加 token；
- 用目标、requirements document 和系统思维减少无效循环。

## 10:00–13:30｜推理扩展：15 倍速度如何改变长时间任务经济学

主持人提出“无限 token = 无限 reasoning”的直觉：现在把模型跑 25–48 小时已可能得到惊人的结果；如果 Cerebras 把速度提高约 **15 倍**，同样跑 24 小时，也许能获得相当于数周或数月的计算/思考推进。

Feldman认同 reasoning 本质上是高强度 inference。模型在内部生成大量 token、分解策略、让多个线程/agent 互相检查，正因为计算量巨大，快速推理机器才有价值。主持人把它与早期 Sam Altman 对 reasoning 的描述联系起来：系统不再只是“猜下一个词、补句子、总结 PDF”，而是理解意图、制定策略并审阅自身工作。

**投资含义**：推理芯片的价值度量不应只看每 token 成本，还要看 time-to-answer、可持续长任务时长、并行 agent 数及“更快速度是否诱发更多 token 消费”。低延迟可能不是单纯降本，而会扩展可行任务边界。

## 13:30–16:30｜Cerebras 如何看自己的性能曲线；客户自研芯片不是“虚张声势”

主持人问 Cerebras 是否有类似 Moore’s Law 的内部目标。Feldman称，传统处理器遵循约每 18 个月翻倍；Cerebras 的 wafer-scale 新架构“打破”了这条旧轨迹。他预计未来 18 个月性能提升会显著超过 2 倍，因为新架构仍有大量围绕实际 workload 的优化空间；而已有约 20 年历史的 GPU 架构更依赖制程缩小和下一 fab node。

主持人随即指出执行风险：公司面对 250 亿美元需求，要同时招聘、部署、迭代硬件和软件，并服务 OpenAI 这样移动极快、要求极高且未来可能自研竞争芯片的客户。

Feldman 的回应不是否认客户自研，而是说当前需求足够大，“没有硅会闲置”。Amazon、OpenAI 等自研芯片也并非仅为向 Nvidia 压价。hyperscaler 曾受制于 Intel/x86，GPU 厂商又曾受制于少数 hyperscaler；各方都不愿完全依赖单一供应商。客户不必造出最快芯片，只需控制自身命运的一部分。

**风险边界**：这一回答承认长期垂直整合是真实趋势。对独立加速器厂商而言，短期 TAM 大并不消除客户集中、内部 ASIC 替代、软硬件生态和部署能力的长期风险。

## 16:30–20:30｜开源模型的“法拉利 / minivan”分层与 AI 主权

主持人称近期使用 Kimi 等开源模型时，部分任务已很难感知与闭源模型的差异；smart routing 进一步降低成本。Feldman用汽车类比回应：困难问题仍需要“法拉利”——OpenAI、Anthropic、Gemini 等 frontier model；但企业里大量工作只是从 Workday 剪切、重新排入表格，这类“cut-and-paste economy”不需要奥数金牌能力，只需要稳定开源模型。

第二个驱动力是数据与智能主权。金融、医疗等受 HIPAA、FINRA 等监管的行业担心数据泄露，也担心把关键能力交给抱负极大的 frontier lab，因此要求本地、境内或 on-prem 部署。

Feldman认为 OpenAI 发布 OSS 12B 是好决定，但美国仍需要更多本土开源模型，否则全球可选项主要是 OSS 12B 或中国模型。Nvidia 也看到了开源机会，但其下游客户包括 OpenAI、Anthropic、xAI、Google，直接推动模型会产生竞合张力。

Cerebras 当前运行的模型包括 GLM、Kimi、Qwen、OpenAI 闭源模型，也为 GSK 运行其自研模型，并支持阿联酋 G42 / MBZUAI 的模型。Feldman以此证明硬件平台要保持模型中立，服务多种主权与企业模型。

## 20:30–27:00｜能力发布、网络安全和监管：先红队、再逐步开放并非不合理

主持人问：若新模型对网络攻击真正危险，政府要求分阶段发布是否合理？Feldman没有判断具体模型或政治过程，而是从原则回答：当模型创造性强到可能构成实质威胁时，政府要求先分阶段推出、给防御方几周红队和修补时间，并非不合理。他反对把流程做成药品审批那种七年周期和沉重文书，但支持在强技术发布前检查 NSA 等关键基础设施是否能防御明显漏洞。

主持人举 Palo Alto Networks 负责人 Nikesh 的例子：把新模型用于自家软件后，一小时内找到数十个此前不知道的关键漏洞，团队据称暂停其他工作、连续修补六周。两人把这与操作系统的多层 beta 类比：消费者能加入公开 beta，但更早还有专门处理安全、数据丢失和数据泄漏的测试层。

Feldman也接受“大规模数据泄漏迟早发生”的现实，用 Buffett 对再保险的说法类比：知道龙卷风或大地震终会发生，只是不知道时间，因此要预先储备和建设韧性。

**投资含义**：前沿模型的 cyber capability 会同时拉动安全产品需求与发布管制。模型发布节奏、政府预览、红队和漏洞修复可能成为 frontier lab 商业化的新门槛，而安全厂商可获得更高频、更机器化的攻防需求。

## 27:00–31:30｜“AGI 已经到来”的口径：按旧定义已越线，但新问题仍未定义

主持人认为，如果按照 20 年前的定义，当前系统已具备 AGI：图灵测试早已被突破，系统能推理、理解意图并在广泛任务上达到人类水平。Feldman基本同意：按 10、20、30 甚至 50 年前提出的测试，今天已经越线。

但这不等于终点。两人的重点是：旧科幻提出的问题已被回答，真正困难的是人类尚未提出的新问题。Feldman以 Ilya 早年强调 AI safety、Elon 早年提出大幅降低火箭发射成本为例：当时听起来边缘，后来变成现实，因此应认真对待能“看过拐角”的人。

## 31:30–40:54｜递归改进、物理世界与丰裕叙事

主持人把 agent 的循环称为 “loop maxing”：工具开始建立工具、多个线程互相检查，形成递归。Feldman把 AI 学习速度与人类跨代学习对比。凡尔赛宫等大型项目曾持续 50、70 或 100 年，由石匠家族三四代传承经验；人类认知范式也常要等一代人退出才改变。

AI 则像果蝇而不是大象：人类或大型哺乳动物的代际跨度约 15–20 年，果蝇可以一天两代；模型迭代让知识在等价于数千代的循环中快速累积。未来瓶颈可能从“智力问题”转成“人的组织问题”：怎样激励团队、减少摩擦、让人协作执行 AI 给出的方案。Feldman提到 world model 观察人类行为后，也许能提供 behavioral insight。

主持人把递归延伸到实体世界：今天让 agent 做一个 Salesforce 替代品，未来也许让一千台 Optimus 在土地上建出一座“凡尔赛宫”。Feldman没有把这当作近期工程预测，而是用来说明过去的宏伟工程对更早时代同样不可想象。

谈到就业和风险，Feldman承认会有经济错位，如汽车替代马车和钉马掌工；但他要求同时列出正面账本：AI 或许能显著降低癌症死亡，带来更充足的能源、食物、知识、教育和住房。他特别以个性化教育为例：人类早知亚里士多德式一对一导师优于面向“中位学生”的工厂化课堂，却因成本无法普及；agent 可按每个孩子的学习方式调整。

他的最终立场不是“没有风险”，而是只要公平地同时评估利弊，账本会明显偏向 abundance。

# 第二部分：Robin Rombach / Black Forest Labs

## 40:54–44:30｜公司与技术底座：latent diffusion 从生成媒体走向 physical AI

Rombach澄清，公司在德国弗赖堡（Black Forest 地区）和旧金山设点，成立约两年，团队此前参与 Stable Diffusion；更早在慕尼黑读博时发明 latent diffusion。

其基本机制是把图片、视频、音频等自然数据压缩到更高效的潜在表示，再在表示上训练 transformer。Rombach把它与 JPEG、MP3 的压缩思想类比，只是实现变成神经网络。该算法成为后续图片、视频生成乃至部分 physical AI 路线的基础。

BFL 现在的目标不只是 text-to-image，而是预训练同时处理图片、视频、音频的视觉多模态模型，再加入 action prediction。同一个模型未来可以生成媒体、理解环境并预测动作，部署到现实机器人。

## 44:30–47:31｜直觉智能与深度推理；生成模型如何摆脱“老虎机”

Rombach把完整智能分成互补层：
- **直觉智能**：从视觉和多模态数据中感知、预测世界；
- **深度推理层**：进行抽象规划和逻辑推理。

BFL 从直觉侧切入。图片计算量低于视频，适合作为起点；视频预训练则隐含学习物理和交互规律，为 action prediction 与机器人铺路。

主持人指出生成模型常像老虎机：给 prompt，返回不可解释结果，用户难以控制风格、颜色和审美。Rombach认为解决路径是把更多 manipulation layer 暴露给用户和开发者：从 text-to-image，进化到“文本 + 图片”的编辑，再到“多张图片 + 文本”的语义组合；视频将沿同一路径发展，最终多种模态同时成为统一模型的输入与输出。

## 47:31–51:30｜与 Martin Scorsese 的合作：先把导演脑中的画面低损耗地表达出来

主持人问何时能生成 Scorsese 愿意发布的完整电影，而非 5–10 秒片段拼成的 AI slop。Rombach先拒绝替创作者定义使用方式：模型是 medium，不应由技术公司告诉 Scorsese 应怎样创作。

他描述了与 Scorsese 同室试用模型的过程。导演脑中有一个东欧村庄的场景，先口述想法，再看模型输出、继续迭代。Scorsese 最后认可的价值是：把头脑中的 mental picture 变成一组可见图片，让团队更容易理解导演真正想要什么。

Rombach认为语言是有损沟通介质，会被不同人重新解释；图像和视频包含更丰富信号。因此，近期价值不是取消导演，而是把创意意图更高保真地外化、加速探索和沟通。

他不确定“一键生成整部长片”是否应成为终极目标。更有意思的用途是 human-in-the-loop：人持续迭代，把模型当新媒体。最好的输出通常来自人和模型的共同工作，而非全自动生成。

## 51:30–55:00｜创意生产的现实降本：storyboard、创业公司发布片、虚拟制景

主持人将其与传统 storyboard 类比：Ridley Scott、Spielberg、George Lucas 都会手绘、使用模型或与概念艺术家长期合作。生成模型可以把 brainstorming 并行化。

在创业公司场景，过去一家融资 1,000 万美元的公司可能花 **10万–25万美元** 制作发布视频；现在部分团队只需与导演合作一两周。Rombach确认 Flux 等基础模型已被用于这类 30–90 秒产品叙事，但强调 BFL 的角色是提供基础模型，让不同产品和工作流在其上构建。

主持人还转述一部约 **3,000 万美元预算** 的电影案例：演员在 sound stage 表演，背景由生成式 AI 完成；若采用传统实景/布景，据称成本可能达 **1.5 亿美元**，项目甚至不会获批。Rombach称已看到类似生产用途，但高端电影仍是最苛刻场景。

技术进步尺度上，他回忆博士阶段只能生成 **64×64** 像素图片，现在已能做高分辨率、多输入视频；轨迹不会停下，但他避免给出“几年内完成整部高质量电影”的硬时间表。

## 55:00–59:30｜同一模型为何既能拍电影又能控制机器人

Rombach最兴奋的不是电影单点市场，而是多模态生成模型的通用性：同一种模型可以生成影片，也可作为机器人 brain。所谓 world model、world action model、生成、模拟和预测，在底层逐渐汇合。

主持人问机器人学习倒水是否要靠第一视角眼镜、手套和高精度动作数据，还是看大量 YouTube 就够。Rombach回答不是二选一，而是 perception、prediction、simulation、generation 和 action prediction 的组合。模型必须理解视觉输入，才能预测合理下一动作、描述场景或把场景转换为新内容。

现实部署仍面临机器人硬件和 action representation 不统一。当前通常先用通用视觉理解，再用针对特定机器人、特定任务的少量数据微调；Rombach称可能只需**数小时 fine-tuning data**。长期目标是更接近 language model 的 in-context control：直接告诉机器人去拿一杯橙汁，无需每台设备重新训练，但他明确称这仍是研究问题。

**投资含义**：BFL 的可选性跨越生成媒体、企业视觉、机器人/physical AI 和客户定制；反过来，这也意味着训练成本、数据权利、硬件异构适配和产品聚焦都会成为执行挑战。

## 59:30–63:56｜IP 主权、Disney 类客户和 BFL 的企业商业模式

主持人问 Disney 等拥有庞大 IP 库的公司应如何使用开源模型。Rombach称，公开工具会禁止生成某些受保护 IP；同时 BFL 会与 IP 持有人共同开发模型，有些基于开源模型，有些基于更强的 proprietary model。

对于 Disney+ 未来形态，他不替客户下结论，但设想随着生成速度提高、交互性增强，流媒体平台可加入互动内容创作工具。主持人进一步提出 fan film 模式：过去 Lucas 允许粉丝在非商业前提下拍 Star Wars 影片；未来 IP 方可按软件租赁、输出或授权收费，让用户用官方角色创作未被讲述的故事。Rombach赞成在保护 IP 方经济利益的同时，为个性化创作找到可行模式。

公司组织方面，Rombach称 BFL **刚超过 100 人**，并已融资，在德国和旧金山继续招聘：
- 有大规模模型训练、diffusion、flow matching 经验的研究员；
- 面向客户共同开发 physical AI 或 IP 定制方案的工程师；
- 管理大规模 compute infrastructure、保证训练稳定并最大化 MFU 的基础设施人才；
- 把技术真正部署到用户手中的产品/商业人才。

主持人的总结是：企业客户既想通过开源获得控制权，又会为更精细能力需要 proprietary frontier model；BFL 可以在开源基础模型、闭源高性能模型和企业定制三层同时变现。
